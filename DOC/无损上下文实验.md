# Lossless Context Experiment

## Goal

Enable and validate the first AstrBot-native `lossless-claw-enhanced` path in `JW-Bot` without changing the default runtime behavior.

This experiment currently includes:

- CJK-aware token counting
- sidecar conversation mirroring
- conservative lossless summary compression
- safe fallback to AstrBot's existing truncation path

## Default Behavior

By default:

- sidecar mirroring is active
- the new token estimator is active
- the new lossless compressor is **not** active

That means current chat behavior remains close to stock AstrBot unless the experimental compressor is explicitly enabled.

## Enable The Experimental Compressor

Set this environment variable before starting AstrBot:

```bash
ASTRBOT_EXPERIMENTAL_LOSSLESS_CONTEXT=1
```

When this flag is enabled:

- the main agent will build a `LosslessSummaryCompressor`
- older history may be compacted into a summary pair
- the latest raw messages will still be preserved
- provider failure will fall back to turn truncation

## What To Validate

Use a long QQ conversation and validate these points:

1. `lossless_context.db` is created automatically.
2. `lossless_items` contains `raw_message` rows for the active conversation.
3. `lossless_heads.last_ingested_seq` increases as the conversation grows.
4. When context pressure is high, the model still sees the latest raw turns.
5. If compression provider calls fail, replies continue normally.

## Log Signals

Expected useful signals in logs:

- sidecar ingest warnings if mirroring fails
- compression start/completion messages from the context manager
- fallback error logs from `lossless_compressor.py` when summarization fails

## Recommended Rollout

Use this order:

1. keep current production runtime unchanged
2. enable the env flag only in a controlled local run
3. inspect `lossless_context.db`
4. verify no regression in QQ replies
5. only then consider adding the flag to the watchdog service

## Not Included Yet

This experiment does **not** yet include:

- DAG condensation
- recall tools like `lcm_expand`
- summary lineage links in live use
- WebUI inspection surfaces

Those belong to later phases.
