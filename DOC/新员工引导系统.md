# 新员工引导系统（Onboarding Guide）

## 概述

`onboarding_guide` 插件在员工首次与机器人对话时自动触发，通过 ABC 选择引导员工完成角色配置，并自动绑定对应的 AI 人格。完成后进入正常使用模式，不再拦截消息。

## 插件位置

```
data/plugins/onboarding_guide/onboarding_guide.py
```

## 工作流程

```
员工发第一条消息（任意内容）
        ↓
系统检测 onboarding_state = None
        ↓
回复欢迎语 + ABC 选项菜单
        ↓
员工回复 A / B / C
        ↓
自动绑定对应人格到该 session
写入 onboarding_state = "completed"
        ↓
后续消息走正常 pipeline（引导不再介入）
```

## 角色与人格映射

| 选项 | 职能方向 | 绑定人格 |
|------|----------|----------|
| A | 推广 / 运营 / 内容 | `Biz_Assistant_Claw` |
| B | 项目管理 / 跟进 | `Enterprise_Ops_Kernel` |
| C | 技术 / 运维 | `DevOps_Console` |

## Session 隔离规则

- 以 `unified_msg_origin`（格式：`{platform_id}:{message_type}:{user_id}`）为隔离单元
- QQ 和飞书各自独立，同一个人在两个平台分别触发一次引导
- 每个员工的对话历史、任务、Memory、人格绑定全部独立

## 状态存储

使用 AstrBot 的 `sp`（session preference）模块，存储在 `data/data_v4.db`：

| Key | 值 | 说明 |
|-----|----|------|
| `onboarding_state` | `"selecting_role"` / `"completed"` | 引导进度 |
| `session_service_config.persona_id` | persona ID 字符串 | 绑定的人格 |

## 管理员操作

### 重置某个用户的引导状态（让其重新选择角色）

```sql
-- 查询用户的 session 配置（在 data/data_v4.db）
SELECT scope_id, key, value FROM preferences
WHERE scope = 'umo' AND scope_id LIKE '%<user_id>%';

-- 清除引导状态（下次发消息重新触发引导）
DELETE FROM preferences
WHERE scope = 'umo' AND scope_id = '<unified_msg_origin>' AND key = 'onboarding_state';
```

### 手动绑定人格（跳过引导直接指定）

```sql
-- 在 data/data_v4.db 中直接写入 session_service_config
-- 注意：value 字段是 JSON 字符串
UPDATE preferences
SET value = '{"persona_id": "Biz_Assistant_Claw"}'
WHERE scope = 'umo' AND scope_id = '<unified_msg_origin>' AND key = 'session_service_config';
```

### 查看某用户当前人格

```sql
SELECT scope_id, value FROM preferences
WHERE scope = 'umo' AND key = 'session_service_config';
```

## 扩展：新增角色选项

在 `onboarding_guide.py` 的 `_CONFIRM` 字典中添加：

```python
_CONFIRM = {
    "A": ("Biz_Assistant_Claw", "推广运营助手", "...提示语..."),
    "B": ("Enterprise_Ops_Kernel", "项目跟进助手", "...提示语..."),
    "C": ("DevOps_Console", "技术运维助手", "...提示语..."),
    "D": ("新人格ID", "新角色名称", "...提示语..."),  # 新增
}
```

同时更新 `_WELCOME` 中的选项说明。修改后复制到 `data/plugins/onboarding_guide/` 并重启 AstrBot。
